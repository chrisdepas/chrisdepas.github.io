RoboCraft Cheat
  • This was a cheat for the free-to-play game Robocraft.
  • The game uses Unity Engine, which uses Mono’s C# Runtime and was protected with Easy Anti-Cheat.
  • Easy Anti-Cheat protects against DLL injection. To bypass this I replaced the Steam API DLL with my own, which exported the same functions, and would call exported Mono functions to load my malicious C# executable.
  • The payload referenced Unity Engine and game libraries to display a simple GUI in-game, and modify the local player’s position and weapon stats.
Advanced CS:GO Cheat
  • This was a cheat for the Counter Strike: Global Offensive
  • The game uses the Source Engine, which was written in C++, and was protected by Valve's VAC2 anti-cheat
  • I was able to gain access to methods and data in the engine by calling exported methods in dlls, in the same way that the game itself does. This was made easier by examining public Source SDK code
  • The Source SDK used appeared to be customized for the game, so I needed to deduce the format of data structures in memory (using ReClass) and function signatures (using OllyDbg). I was also able to get more information by examining the Mac OS binaries in a decompiler (IDA), which contained unobfuscated symbols
  • I gained access to game data by first finding it in memory (using the memory scanner MHS), and then tracing this back to a base pointer accessed by the code somewhere. I was then able to create a unique signature of the bytes in the function, which I could search for and use to dynamically retrieve game data pointers
  • Using these techniques, I was able to create a fully-featured cheat, with an in-engine menu, ray-tracing capability, texture removal (i.e. 'No Hands'), and many other features
